# PPT 演示优化专家 (ppt-optimizer)

## 📖 简介

这是一个专为 Aone Copilot 设计的 Skill，旨在帮助用户全方位优化 PPT 演示文稿。无论是业务汇报、融资路演、技术培训还是学术答辩，该 Skill 都能从**内容逻辑、视觉设计、演讲适配**三个维度提供专业的诊断与优化建议，显著提升信息传递效率与专业度。

## ✨ 核心功能

-   **故事线重构**：按"问题→方案→价值→行动"梳理叙事结构，确保逻辑连贯
-   **单页单主题**：识别信息过载页面，建议拆分或聚焦核心观点
-   **标题即结论**：将描述性标题转化为观点型标题，提升信息密度
-   **视觉规范检查**：验证对齐、留白、字体层级、配色对比度、图表简化
-   **演讲适配增强**：补充备注、预估时长、标记互动节点、准备备用页
-   **分级诊断报告**：按 🔴高/🟡中/🟢低 优先级输出问题清单与修改对比

## 🚀 快速开始

### 1. 安装 Skill
将本目录复制到你的项目或全局 Skill 目录中：
```bash
# 项目级（仅当前项目可用）
cp -r ppt-optimizer /your-project/.aone_copilot/skills/

# 全局级（所有项目可用）
cp -r ppt-optimizer ~/.aone_copilot/skills/
```

### 2. 使用方式
在 Aone Copilot 对话中直接输入以下指令即可触发：

-   "帮我优化这份 PPT"
-   "润色一下这个汇报演示"
-   "检查 PPT 逻辑是否清晰"
-   "让路演材料更有说服力"
-   直接上传 `.pptx` / `.ppt` / `.key` 文件并附带优化需求

### 3. 文件支持说明
-   **支持格式**：`.pptx`（推荐，完整解析）；`.ppt`（旧版格式，仅提取文本与基础结构，动画/母版信息可能丢失）；`.key`（Keynote 文件需用户先导出为 `.pptx` 再上传，Copilot 不直接解析 .key）
-   **文件大小**：建议 ≤50MB；超过 100MB 的文件可能因解析超时失败，建议压缩图片或拆分文件
-   **加密文件**：不支持密码保护的 PPT，请先解除加密再上传
-   **嵌入媒体**：视频/音频文件不会被解析，仅检查占位符位置与标注；SmartArt 会转为静态图片处理

### 4. 输出示例
Skill 将返回结构化的诊断报告，包含：
-   演示类型、目标受众、时长预估
-   整体评分（0-100）
-   分级问题清单（含页码定位）
-   具体的修改前后对比
-   可选的完整文案重写版本

## 📋 适用场景

| 场景 | 优化重点 | 典型问题 |
|------|----------|----------|
| 高管汇报 | 结论前置、数据精炼、行动明确 | 标题模糊、细节过多、无决策点 |
| 融资路演 | 故事感、市场验证、团队亮点 | 技术术语堆砌、缺乏用户证言 |
| 技术培训 | 步骤可视化、知识分层、实操引导 | 文字密集、缺少示意图、无练习环节 |
| 学术答辩 | 逻辑严谨、证据充分、问答预案 | 论点跳跃、引用不规范、超时风险 |
| 产品发布 | 情绪调动、卖点聚焦、视觉冲击 | 功能罗列、缺乏场景、动画杂乱 |

## ⚙️ 自定义配置

如需调整优化侧重点，可编辑 `SKILL.md` 中的以下部分：

-   **角色定位**：修改为特定领域专家（如"医疗学术演示顾问"、"ToB 销售演示教练"）
-   **核心能力**：增删特定维度的检查项（如增加"多语言适配"、删除"动画克制"）
-   **输出格式**：调整报告模板以匹配团队汇报规范
-   **触发条件**：添加更多激活关键词或文件类型

## 📐 设计规范参考来源

本 Skill 推荐的视觉参数均基于行业公认标准与权威指南，非主观假设：

-   **留白边距 ≥1cm**：源自微软 Office 官方设计指南《PowerPoint Design Best Practices》及 Garr Reynolds《Presentation Zen》（2008），确保投影裁切安全区与视觉呼吸感
-   **字体层级 3:2:1**：基于 Richard E. Mayer《Multimedia Learning》（Cambridge University Press, 2009）中的"信号原则"（Signaling Principle），标题/正文/注释的显著差异可降低外在认知负荷
-   **WCAG AA 对比度 ≥4.5:1**：W3C《Web Content Accessibility Guidelines 2.1》（2018）强制标准，适用于所有公开演示材料，确保色觉障碍者可读
-   **正文最小 18pt**：来自 TED Talks 官方演讲者手册《TED Speaker Guide》（2023）及苹果 Keynote 设计指南，保证 10 米外后排观众可读性
-   **每页 1-2 分钟**：基于 John Sweller《Cognitive Load Theory》（Psychology of Learning and Motivation, 2011）及麦肯锡《The Pyramid Principle》（Barbara Minto, 2009）汇报实践，超过 2 分钟/页通常意味着信息过载或逻辑不清

## ⚠️ 注意事项

1.  **版权合规**：优化建议中涉及的图片、字体、图标需自行确认商用授权，Skill 不提供素材
2.  **数据准确性**：所有优化仅针对表述与呈现，不修改原始数据，请务必核对源文件
3.  **设备测试**：颜色、动画效果在不同投影仪/屏幕上表现差异大，正式演示前务必实地测试
4.  **无障碍优先**：建议为图表添加替代文本、为视频添加字幕，符合包容性设计原则
5.  **大文件处理**：超过 30 页的 PPT 建议分章节提交，避免上下文丢失导致建议碎片化
